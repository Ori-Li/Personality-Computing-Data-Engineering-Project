# 文化人格计算数据集工程策略指南

# Cultural Personality Computing Dataset Engineering Strategy Guideline

## 1. 项目目标 Project Objective（项目目标）

本项目旨在构建一个全球文化知识图谱（Cultural Knowledge
Graph，文化知识图谱），将人物、作品、虚拟角色、媒体资产以及心理体验向量连接起来。

最终目标：

文化实体 → 文化作品 → 内容体验 → 心理向量 → 人格偏好模型

数据集不是静态名单，而是一个可以持续扩展的文化认知系统。

------------------------------------------------------------------------

# 2. 数据总体架构 Data Architecture（数据架构）

整体数据结构：

    Entity Layer（实体层）
            ↓
    Work Layer（作品层）
            ↓
    Relationship Layer（关系层）
            ↓
    Media Asset Layer（媒体资产层）
            ↓
    Psychological Vector Layer（心理向量层）
            ↓
    Personality Preference Layer（人格偏好层）

------------------------------------------------------------------------

# 3. 实体层 Entity Layer（实体数据层）

Entity Layer 负责存储文化主体。

包括：

-   Real Human Entity（真实人物实体）
-   Fictional Character（虚拟角色）
-   Group Entity（团体实体）
-   Creative Team（创作团队）

------------------------------------------------------------------------

## 人物建设流程 Character Construction Pipeline（人物建设流程）

    领域人物名单
    ↓
    实体验证 Entity Verification
    ↓
    名称标准化 Name Normalization
    ↓
    实体消歧 Entity Resolution
    ↓
    职业扩展 Career Expansion
    ↓
    作品关系建立 Work Relationship Building

------------------------------------------------------------------------

# 4. 人物数据规则 Character Data Rules（人物数据规则）

输入名单只是候选来源，不代表全部正确。

必须验证：

-   是否真实存在
-   身份是否正确
-   别名
-   原语言名称
-   职业
-   文化贡献
-   公开作品

禁止：

-   虚构人物
-   无可靠信息人物
-   错误匹配
-   编造作品

------------------------------------------------------------------------

# 5. 人物心理资料生成 Psychological Biography（心理相关人物资料）

人物介绍不应只是百科简介。

应包含：

-   创作理念 Creative Philosophy
-   艺术风格 Artistic Style
-   生涯选择 Career Decisions
-   重要经历 Life Experience
-   创作方法 Creative Method
-   长期行为模式 Long-term Patterns

目标：

通过事实材料推断可能的：

-   思维方式
-   价值取向
-   创造动力
-   审美偏好

禁止直接赋予人格类型。

------------------------------------------------------------------------

# 6. 作品层 Work Layer（作品数据层）

作品是文化计算的核心载体。

支持：

-   电影 Film
-   电视剧 TV Series
-   游戏 Game
-   小说 Novel
-   音乐 Music
-   漫画 Manga
-   动漫 Animation
-   绘画 Painting
-   建筑 Architecture
-   摄影 Photography
-   舞台 Stage Performance
-   雕塑 Sculpture

------------------------------------------------------------------------

## 作品数据结构 Work Schema（作品结构）

包括：

-   workId（作品唯一ID）
-   type（作品类型）
-   subcategory（分类）
-   countryName（国家）
-   year（年份）
-   workName（中文名称）
-   originalName（原始名称）
-   creators（创作者）
-   introduction（作品介绍）
-   culturalImpact（文化影响）

------------------------------------------------------------------------

# 7. 人物-作品关系 Person-Work Relationship（人物作品关系）

核心关系：

    Person
     ↓
    created / performed / contributed
     ↓
    Work

包括：

-   导演 Director
-   编剧 Writer
-   演员 Actor
-   作曲家 Composer

-   建筑师 Architect

------------------------------------------------------------------------

数据不是单向生成。

循环：

    人物
    ↓
    作品
    ↓

    ↓

    ↓

例如：

电影：

导演、演员、摄影、音乐制作人

游戏：

## 虚拟角色扩展 Fictional Character Expansion（虚拟角色扩展）

    ↓

# 11. 图片评估 Image Evaluation（图片评估）

采用多阶段策略：

    规则评分
    ↓
    Embedding Matching
    ↓
    视觉质量评分
    ↓
    异常审核

------------------------------------------------------------------------

# 12. 作品心理向量层 Psychological Vector Layer（心理向量层）

目标：

分析作品带来的心理体验。

不是判断作者人格。

------------------------------------------------------------------------

## 向量维度 Vector Dimensions（向量维度）

包括：

### Semantic Vector（语义向量）

主题、关键词、文化意义。

### Experience Vector（体验向量）

例如：

-   治愈
-   紧张
-   敬畏
-   怀旧
-   孤独

### Cognitive Vector（认知向量）

例如：

-   抽象程度
-   复杂度
-   思考深度

### Emotion Vector（情绪向量）

例如：

-   快乐
-   悲伤
-   恐惧
-   希望

### Aesthetic Vector（审美向量）

例如：

-   极简
-   黑暗
-   梦幻
-   实验性

------------------------------------------------------------------------

# 13. 媒介专属向量 Media Specific Vector（媒介专属向量）

不同媒介使用不同分析维度。

## 游戏 Game

-   玩家自主性 Player Agency
-   探索深度 Exploration
-   系统复杂度 System Depth

## 音乐 Music

-   节奏 Tempo
-   能量 Energy
-   情绪 Emotion
-   乐器结构 Instrumentation

## 建筑 Architecture

-   空间体验 Spatial Experience
-   象征意义 Symbolic Meaning

------------------------------------------------------------------------

# 14. 人格关联 Personality Preference Model（人格偏好模型）

不是：

作品 = 某人格

而是：

    作品体验向量
    ↓
    认知偏好
    ↓
    人格内容匹配

可映射：

-   Ni
-   Ne
-   Ti
-   Te
-   Fi
-   Fe
-   Si
-   Se

以及其他人格维度。

------------------------------------------------------------------------

# 15. 数据库设计 Database Design（数据库设计）

核心表：

## Entity Table（实体表）

保存人物、团体、角色。

## Work Table（作品表）

保存文化作品。

## Relationship Table（关系表）

保存：

人物-作品关系。

## Media Asset Table（媒体资产表）

保存图片和来源。

## Vector Table（向量表）

保存心理和内容向量。

------------------------------------------------------------------------

# 16. 自动化系统 Automated Pipeline（自动化流程）

LLM（大语言模型）：

负责：

-   语义理解
-   信息抽取
-   关系发现

Embedding Model（向量模型）：

负责：

-   相似度
-   聚类
-   图像匹配

Rule System（规则系统）：

负责：

-   去重
-   验证
-   格式检查

Human Review（人工审核）：

负责：

-   高价值异常

------------------------------------------------------------------------

# 17. 数据质量原则 Data Quality Principle（数据质量原则）

优先级：

    正确性 Accuracy
    >
    实体验证 Verification
    >
    完整性 Completeness
    >
    数量 Quantity

禁止：

-   虚构实体
-   虚构作品
-   无证据关系

------------------------------------------------------------------------

# 18. 最终目标 Final Goal（最终目标）

构建：

    全球文化知识图谱
    +
    多模态文化数据库
    +
    作品心理体验空间
    +
    人格内容计算模型

形成可持续增长的文化智能系统。
