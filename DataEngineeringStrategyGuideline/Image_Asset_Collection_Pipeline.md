# 人物与作品图片资产采集 Pipeline 设计

## 1. 总体目标

建立一个可扩展的文化实体图片资产系统：

实体（Entity） → 作品（Work） → 图片候选池（Image Candidates） →
多模型评估 → 主封面选择 → 对象存储 + 数据库关联

目标不是简单下载图片，而是建立可持续更新的多源媒体资产库。

------------------------------------------------------------------------

# 2. 核心数据流程

    实体数据库
        ↓
    实体解析 Entity Resolution
        ↓
    生成图片搜索任务
        ↓
    多数据源采集
        ↓
    图片候选池
        ↓
    OCR / Embedding / 质量评估
        ↓
    置信度评分
        ↓
    选择 Cover
        ↓
    对象存储
        ↓
    数据库关联

------------------------------------------------------------------------

# 3. 实体数据要求

不要直接使用名字搜索：

``` json
{
"name":"周深"
}
```

应该：

``` json
{
"id":"person_000001",
"name":"周深",
"type":"person",
"domain":"music",
"country":"China",
"aliases":[
"Zhou Shen"
],
"works":[
"大鱼"
]
}
```

图片系统依赖 entity_id，而不是 name。

# 5. 使用模型

## 5.1 OpenCLIP / SigLIP

用途：

图文语义匹配。

判断：

    图片是否符合实体

例如：

文本：

    Genshin Impact official game cover

图片：

游戏宣传图

输出：

    similarity = 0.92

------------------------------------------------------------------------

## 5.2 PaddleOCR

用途：

识别：

-   海报标题
-   游戏 Logo
-   专辑名称
-   书名

提高实体匹配。

------------------------------------------------------------------------

## 5.3 图片质量模型

推荐：

-   NIMA
-   CLIP aesthetic score

判断：

-   清晰度
-   构图
-   展示价值

------------------------------------------------------------------------

## 5.4 图片分类

使用 CLIP Zero-shot：

分类：

    poster
    portrait
    screenshot
    painting
    building
    logo

------------------------------------------------------------------------

## 5.5 去重

使用：

-   pHash
-   Image Hash

检测重复图片。

------------------------------------------------------------------------

# 6. 图片评分策略

最终评分：

    Final Score =
    来源权威度
    +
    实体匹配度
    +
    视觉语义匹配
    +
    图片质量
    +
    元数据匹配

示例：

``` python
score =
source * 0.30 +
clip * 0.30 +
ocr * 0.15 +
quality * 0.15 +
resolution * 0.10
```

------------------------------------------------------------------------

# 7. 自动审核规则

## 高置信度

    score >= 0.9

自动作为封面。

------------------------------------------------------------------------

## 中置信度

    0.7 <= score < 0.9

进入抽检。

------------------------------------------------------------------------

## 低置信度

    score < 0.7

删除或重新搜索。

# 12. 设计原则

1.  不使用名字直接绑定图片。
2.  图片绑定 entity_id。
3.  多源采集，不依赖单一网站。
4.  模型用于排序，不用于完全替代规则。
5.  LLM只处理低置信度异常。
6.  图片资产和实体数据分离。
