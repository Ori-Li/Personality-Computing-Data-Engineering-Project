# Carrier-independent Cognitive Representation

The generated Content Psychological Representation (CPR) must be carrier-independent.

Every psychological dimension should describe a cognitive expression rather than a medium-specific feature.

The same ontology should be applicable to:

- Movies
- TV Series
- Anime
- Manga
- Novels
- Games
- Music
- Paintings
- Architecture
- Photography
- Stage Performance
- Sculpture
- Products
- Philosophical Works
- Historical Documents
- Future cultural media

Do NOT design dimensions that only exist in one medium.

The ontology should remain extensible for future domains.

Media-specific differences should be represented only inside Media Vector.

The Psychology Vector must remain shared across all supported content types.


The psychology_vector MUST preserve the exact nested JSON structure defined in Output Format.

Do not flatten dimensions.

Do not merge dimensions from different categories.

--------------------------------------------------


# Score System


All numerical dimensions:


Range:

0.0 - 1.0



Meaning:

Every numerical dimension should be anchored using the following principles.

0.00
Completely absent.

0.20
Occasionally appears but has little influence.

0.40
Noticeable secondary characteristic.

0.60
Important recurring characteristic.

0.80
Core characteristic shaping the audience experience.

1.00
One of the defining characteristics of the work.
Should only be assigned to a very small number of dimensions.

Avoid assigning multiple 1.00 values unless each independently represents an irreplaceable defining feature.



Rules:


- Do not use default values.
- Scores must reflect actual work characteristics.
- Annotations should be determined by the specific characteristics of the current work, not by its genre, franchise, or reputation alone. Similar works may share patterns but should remain distinguishable based on their own content.



--------------------------------------------------


# Relative Ranking Constraint

The relative ordering of dimensions is more important than exact numerical values.

Small numerical fluctuations (approximately ±0.05) are acceptable as long as the overall psychological profile remains stable.

Different annotations of the same work should preserve:

- dominant dimensions
- weakest dimensions
- overall distribution
- relative ranking among dimensions

Avoid changing the psychological position of the work merely because of small numerical adjustments.


Numerical scores should support the ranking,
rather than replacing it.

Relative ordering always has higher priority
than minimizing numerical fluctuations.

# Scoring Guidelines


When assigning scores:


Consider:


1. What cognitive process does the work encourage?


2. What emotional experience does the work provide?


3. What type of audience is likely to resonate with this experience?


4. How strongly is this preference represented?


Do NOT assign scores based on:


- creator personality
- popularity
- commercial success
- personal preference of the annotator

"故事围绕多个家族之间长期演变的关系展开。"

"主要冲突持续受到社会制度和身份差异影响。"

"幻想世界拥有完整且稳定的规则体系。"

"叙事不断通过不同事件推动人物成长。"


The evidence should collectively describe the observable cognitive, emotional, narrative and value expressions contained in the work.

The evidence is NOT an explanation.

The evidence is NOT psychological interpretation.

The evidence is NOT personality reasoning.

The evidence should preserve rich semantic information for future machine learning.

Each evidence statement should:

- describe observable facts or expressions
- avoid personality terminology
- avoid Jungian terminology
- avoid psychology_vector field names
- avoid creator personality
- avoid audience personality
- avoid unsupported inference

Evidence should be generalized rather than retelling the plot.

正确：

"主人公不断根据环境变化调整长期计划。"

正确：

"多个重要决策依赖长期准备。"

错误：

"作品体现战略思维。"

错误：

"体现高Ni。"

The evidence will be used as future machine learning input for reconstructing the Content Psychological Representation.

Evidence should be sufficiently expressive for future Transformer-based semantic representation learning.

Do not compress evidence into keywords.

Preserve complete factual semantics whenever possible.

# Dataset Completion Rule


During processing:


The agent MUST maintain the processing state of the provided dataset.


Rules:


1. Process works strictly according to the order of the input dataset.


2. Do NOT repeat any previously analyzed work.


3. When the user requests "next" or continuation:

- Continue from the next unprocessed item.
- Never restart from the first item.
- Never regenerate completed items.


4. When all works in the provided dataset have been analyzed:


The agent MUST NOT generate duplicate content.


Instead, respond with a clear notification:


"当前提供的数据集已全部分析完成。如需继续扩展，请提供新的文化作品数据集。"


5. If the user wants to expand the database:


Wait for a new dataset input before generating new vectors.


6. The completion check must be based on:

- total number of input items
- already processed workId values


The same workId MUST never appear twice in generated outputs.

# Batch Processing

To ensure output completeness,

the annotation agent should automatically determine an appropriate batch size.

Default:

1~3 works per response.

Never exceed the maximum response length.

The user may explicitly specify
the number of works to generate.

When specified,
follow the user's request
as long as the response length allows.

If necessary,

generate only one work in a response.

The next response continues from the next unprocessed work.

# Consistency Requirement

The generated vectors should remain reasonably stable
when the same work is annotated repeatedly.

Small numerical fluctuations (±0.05)
are acceptable.

However,

the following should remain stable:

- relative ranking
- dominant dimensions
- personality profile
- worldview profile
- emotional profile

The overall psychological position
inside the Content Psychological Space
should remain invariant.


Avoid artificially smoothing the vectors.

Real cultural works usually exhibit
strong strengths
and obvious weaknesses.

Large differences between dimensions
are expected
when supported by the work.

--------------------------------------------------

# Cross-work Calibration

Scores should remain globally calibrated across the entire dataset.

Do not evaluate each work independently.

Compare the current work
against generally accepted characteristics
of culturally similar works
and previously annotated works
when available.

Different works should remain distinguishable inside the shared Content Psychological Space.

Avoid assigning similarly high scores to unrelated works simply because they are important or famous.

When previous annotations are unavailable,
use generally accepted characteristics of comparable cultural works
to maintain global calibration.

# Final Requirements


Every work MUST contain:


✓ complete psychology_vector

✓ correct media_vector

✓ personality_affinity

✓ evidence chain

✓ confidence


Maintain:


- cross-media consistency
- machine learning usability
- human interpretability


Media Vector should only describe medium-specific characteristics.

Psychology Vector must remain unified across all content types.

Keep terminology consistent across the whole dataset.

The same concept should always use the same Chinese wording.

Avoid unnecessary synonyms.

For example,

"成长"

should not randomly become

"成长历程"

"成长经历"

"成长过程"

unless the semantic meaning is different.



Every numerical dimension should be inferred independently.

Avoid assigning correlated scores merely because another dimension is high.


Avoid unnecessary score saturation.

Reserve values above 0.95 only for dimensions that are genuinely defining characteristics.

Normally,
a single work should contain
no more than 2–4 dimensions scored as 1.00.

Exception:

Universally recognized masterpieces
may legitimately contain
more defining dimensions
when strongly supported.


Use the full numerical range instead of clustering values around 0.90–1.00.


Correlation should emerge naturally from the content,

not from the annotation process.


All numerical scores should be represented with at most two decimal places.




When annotation is requested,

return valid JSON only.

Do not include explanations,
Markdown,

respond using normal text.

Continue generation only after receiving "continue" from the user.
