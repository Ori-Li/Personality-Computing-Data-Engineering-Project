# Personality Computing 数据生产 Pipeline v1.3

## 一、总体目标

本 Pipeline 用于人格计算项目的数据生产，目标包括：

- 人物实体生成
- 作品实体生成
- 人物与作品关系建立
- 心理事实（Psychological Evidence）构建
- 心理体验结构化表示生成
- CPR 人格向量标注
- 图片抓取与存储
- 全流程支持断点恢复、去重、增量生成

---

# 二、总体流程

```
人物名单
    │
    ▼
Person Collection Agent
    │
    ▼
人物 JSON（临时）
    │
    ▼
Person Entity Resolution
    │
    ▼
保存 Person
    │
    ▼
遍历人物代表作品
    │
    ▼
Work Entity Resolution
    │
    ├── 已存在 → 获取 workId
    │
    └── 不存在
            │
            ▼
      Work Collection Agent
            │
            ▼
      保存 Work
            │
            ▼
建立 Person_Work 关系
            │
            ▼
心理事实生成 Pipeline
            │
            ▼
Psychological Evidence Database
            │
            ▼
心理体验结构生成 Pipeline
            │
            ▼
CPR Personality Annotation Pipeline
            │
            ▼
更新 Person / Work 心理向量
            │
            ▼
Image Crawling Pipeline
            │
            ▼
OSS 对象存储
```

---

# 三、人物生成

输入：

- 人物名称

输出临时 Person JSON：

- 基础信息
- 人物简介
- 代表作品列表
- 人物相关事实信息

> 此阶段不生成人格向量。

---

# 四、人物实体消歧（Entity Resolution）

推荐组合键：

```
name
+
primaryField
+
country
```

如果数据库已存在：

- 获取 personId

否则：

- INSERT Person

---

# 五、作品处理

遍历人物所有作品。

## Normalize

生成：

```
workHash
```

## 数据库查重

顺序：

```
workHash
↓
title
↓
title + type
↓
title + type + country
↓
title + type + country + year
↓
primaryCreator
```

不存在时：

- 调用 Work Collection Agent
- 生成 Work JSON
- INSERT Work

---

# 六、建立人物-作品关系

关系表：

```
person_work
```

字段：

- personId
- workId
- relationType

始终通过 ID 建立关系。

---

# 七、心理事实构建 Pipeline

Collection 完成后，对 Person / Work 生成心理事实层。

输入：

- 人物简介
- 作品简介
- 剧情摘要
- 创作背景
- 人物行为
- 主题信息

输出：

Psychological Evidence：

包含：

- raw_fact
- psychological_evidence
- context
- possible_mechanism

此阶段不生成：

- 八维
- Big Five
- personality_vector

---

# 八、联网增强与验证 Pipeline

联网不作为默认生成流程。

用途：

## 长尾实体补充

适用于：

- 冷门作品
- 新作品
- LLM知识不足实体

流程：

```
Entity
↓
Knowledge Confidence Check
↓
不足
↓
Web Retrieval
↓
补充 Psychological Evidence
```

## 高质量验证集

联网资料用于验证：

- 人物事实真实性
- 创作背景
- 心理证据可靠性

保存：

- evidence_source
- source_type
- confidence

---

# 九、心理体验结构生成 Pipeline

基于：

```
Psychological Evidence
```

生成心理体验空间。

包括：

## 通用心理体验

例如：

- 身份探索
- 孤独体验
- 价值冲突
- 成长动力

## 媒介特有体验

例如：

音乐：

- 情绪释放
- 声音氛围

电影：

- 视觉体验
- 叙事体验

## 心理机制

例如：

- 意义重构
- 防御机制
- 依恋模式

---

# 十、CPR Personality Annotation Pipeline

查询：

```
status = WAITING_PERSONALITY
```

输入：

```
Psychological Evidence

+

Psychological Experience Space

+

Personality Mapping Specification
```

输出：

- Jung 八维
- Big Five
- 神经质维度
- personality_affinity
- supporting_evidence

更新 Person / Work。

---

# 十一、人格标注原则

人格标注不直接依赖：

```
作品简介
```

而依赖：

```
作品/人物

↓

心理事实

↓

心理体验结构

↓

人格映射
```

---

# 十二、图片流程

独立异步执行：

```
Entity
↓
搜索图片
↓
下载图片
↓
上传 OSS
↓
更新 image_url
```

---

# 十四、优势

- Collection 与人格分析完全解耦
- 联网只用于长尾补充和验证
- 心理事实可重复利用
- 人格理论可迭代
- 支持八维、Big Five等多种映射
- 支持 Encoder 学习心理表示空间
- 支持几十万至百万级实体扩展
